## Field Descriptions

- **Bug ID / ID Number**: Unique identifier for the bug (e.g., BUG-001, BUG-002)
- **Bug ID / Title**: Short, descriptive title of the bug
- **Bug ID / Reporter**: Name of the person who reported the bug
- **Bug ID / Submit date**: Date the bug was reported (YYYY-MM-DD format)
- **Bug overview / Summary**: Brief description of what the bug is
- **Bug overview / URL**: The URL where the bug occurs
- **Bug overview / Screenshot**: Reference to screenshot file or attach image
- **Environment / Platform**: Platform where bug occurs (Web, Mobile, Desktop, etc.)
- **Environment / Operating System**: OS version (Windows 11, macOS 14.2, iOS 17.2, etc.)
- **Environment / Browser**: Browser and version (Chrome 120, Safari 17.2, etc.)
- **Bug details / Steps to reproduce**: Numbered steps to reproduce the bug
- **Bug details / Expected result**: What should happen when following the steps
- **Bug details / Actual result**: What actually happens (the bug behavior)
- **Bug tracking / Severity**: Bug severity level (Critical, High, Medium, Low)
- **Bug tracking / Assigned to**: Person or team responsible for fixing the bug
- **Bug tracking / Priority**: Priority level (P0, P1, P2, P3, etc.)
- **Notes / Notes**: Any additional information, workarounds, or observations
